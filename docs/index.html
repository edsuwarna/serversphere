Manage semua VPS kamu dari satu dashboard

Punya 5, 10, atau 50 server tersebar di berbagai provider?
Stop jumping between SSH tabs. Semua dari browser, tanpa install client.
📖 Documentation GitHub →

Ini dimulai dari masalah gue sendiri: manage 15+ VPS di Linode, Vultr, DigitalOcean, sama local server — buka SSH satu-satu, inget IP mana buat apa, kasih akses ke tim tapi gak mau mereka punya akses full ke semua server.

ServerSphere lahir dari situ. Dashboard yang ngelakuin SSH, container management, monitoring, dan RBAC — semua dari browser, tanpa agent atau tools tambahan di VPS.

💡 Cara kerjanya

🔌

Connect via SSH

ServerSphere pake Paramiko buat SSH ke VPS kamu. Gak perlu install agent — cukup SSH key kayak biasa.

📡

Data via SSH

Semua data — status server, container, logs, metrics — diambil real-time lewat koneksi SSH. Satu port, zero config.

🌐

Dashboard di browser

Semua tersaji di web UI. Terminal interaktif, container controls, monitoring graphs — tinggal klik.

✨ Fitur

Gak usah SSH cuma buat ngecek uptime atau restart container

🗺️

VPS Overview

Semua server dalam satu halaman. Status online/offline, IP, resource usage — langsung kelihatan mana yang merah.

💻

SSH Terminal

Terminal beneran via WebSocket (xterm.js). Bisa apt update, top, vim — dari browser. Buka banyak tab, beda server.

🐳

Container Management

List, start, stop, restart, hapus container dari dashboard. Lihat logs & resource usage per container. Gak perlu docker ps tiap kali.

📋

System Logs

Akses syslog, auth, kernel, docker, nginx logs langsung. Filter by severity, cari keyword. No more tail -f SSH sessions.

📊

Resource Monitoring

CPU bar, RAM used/total, Disk root, Load Average. Visual progress bars, real-time updates.

🔐

RBAC + User Management

Admin, Operator, Viewer — per VPS access control. Kasih akses tim tanpa kasih SSH key. Audit trail semua aktivitas.

🎯 Cocok buat

🧑‍💻 DevOps

Manage 5-50 VPS dari satu dashboard. Gak perlu buka terminal 50 kali.

👨‍🔧 Sysadmin

Monitor, terminal, container ops tanpa install client di laptop masing-masing.

👥 Team Lead

Kasih akses terbatas per VPS ke anggota tim. Lihat siapa ngapain aja via audit log.

🚀 Jalanin 2 menit

Docker image siap pakai, tinggal docker compose up -d

Pakai Image
Clone & Build
# docker-compose.yml — image dari GHCR, gak perlu clone repo
services:
  serversphere:
    image: ghcr.io/edsuwarna/serversphere:latest
    container_name: serversphere
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      - DASHBOARD_PASS=ganti-ini
      - SECRET_KEY=ganti-ini-juga
      - POSTGRES_PASSWORD=ganti-ini
    volumes:
      - ~/.ssh:/root/.ssh:ro
    depends_on:
      serversphere-db:
        condition: service_healthy
  serversphere-db:
    image: postgres:18-alpine
    container_name: serversphere-db
    restart: unless-stopped
    environment:
      - POSTGRES_PASSWORD=ganti-ini
    volumes:
      - pg-data:/var/lib/postgresql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U vpsadmin -d vpsdashboard"]
      interval: 5s
      timeout: 5s
      retries: 5
   volumes:
      pg-data:

Jalankan: docker compose up -d — buka http://server-ip:8080

# Clone repo & build sendiri
git clone https://github.com/edsuwarna/serversphere.git
cd serversphere

# Build & jalanin
docker compose up -d --build
